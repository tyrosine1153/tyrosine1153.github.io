# Commit Message Rule

제목을 간단히 쓰고 본문에 자세히 써도 되니까 다른 메세지와 통일감있게 쓰기

내용이 다른 파일끼리는 반드시 커밋을 나누기

### 제목

- 첫번째 : 어떻게 했나요, 첫글자 대문자
  - `Add`, `Edit`, `Delete`



- 두번째 : 무엇을 했나요, 두번째까지는 비슷한 내용을 한 단어로 뭉치기
  - `posts`, `settings`, `manual`
  - posts 폴더 내에 있는 모든 일은 `posts`로 하기
  - 블로그 설정(config, favicon)들은 모두 `settings`로 하고 자세한 내용을 세번째에 쓰기
  - readme나 설명서 등은 `manual`



- 세번째 : 왜 했나요, to ~
  - `to fix typo`, `to solve error` 



### 본문

무엇을 했는지, 왜 했는지에 대한 자세한 설명. 한국어로!
